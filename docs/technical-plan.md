# Technical Plan

## Goal

Build a restrained, bilingual B2B website for Alaska Film & TV that makes the company credible to commissioning editors, public authorities, procurement officers, corporate communication teams, press and industry buyers.

## Architecture

- Astro static site for high performance and low operational complexity.
- TypeScript data model in `src/data/site.ts` as an interim content layer.
- Future CMS can map to the same fields without changing page templates.
- Tailwind CSS for design tokens and component styling.
- Static routes plus generated production pages.
- Sitemap generated by `@astrojs/sitemap`.

## CMS Recommendation

Sanity is the preferred next step because editorial teams can work with structured content, asset metadata and role-based workflows without needing a full application backend. Payload is also viable if the team wants to self-host and own the backend.

## Performance Targets

- Static output by default.
- No heavy frontend framework runtime.
- No autoplay video.
- Local or optimized CMS images once approved assets exist.
- Target Lighthouse score: 95+ across categories.
- Target mobile LCP: below 2.0 seconds with optimized hero and production images.

## Accessibility Targets

- Semantic sections, headings and lists.
- Keyboard-visible focus states.
- High contrast black/off-white base.
- No essential information hidden behind hover.
- Contact routes available as direct mail and phone links.

## SEO

- Organization JSON-LD on all pages.
- TVSeries JSON-LD on production detail pages.
- Hreflang between Swedish and English pages.
- OpenGraph and Twitter card metadata.
- Sitemap and robots file.

## Analytics

Use Plausible or Fathom. Keep analytics cookieless and avoid consent banners unless future embedded media or marketing scripts require them.
