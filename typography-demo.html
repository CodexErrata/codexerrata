Typography Demonstration

A showcase of all typographic elements

Paragraph Indentation Rules

This is the first paragraph after a heading. According to our CSS rules, it has NO indent (text-indent: 0). This creates a clear visual break between sections and helps establish hierarchy.

This is the second paragraph. Notice how it IS indented by 1.5em. All subsequent paragraphs follow this pattern, creating the classic book-style typography that Tufte's work exemplifies. The indent signals a new paragraph without requiring extra vertical space.

Third paragraph, also indented. The consistent indentation creates a rhythm that guides the eye down the page. Combined with justified text and careful line-height, this produces an even "color" across the page—a key goal in classical typography.

After a Level 3 Heading

Again, no indent for the first paragraph. This rule applies to all heading levels (h1 through h6), ensuring consistency throughout the document.

And back to normal indentation for subsequent paragraphs.

The Blockquote Exception

Here's a paragraph before a blockquote, with standard indentation.

This is a blockquote. It's styled in italics with a left border. Blockquotes can contain multiple paragraphs.

Notice that paragraphs within blockquotes have no indent, maintaining clarity within the quoted material.

The abstract box provides a special container for abstract sections, with a light gray border and subtle background.

Editorial Box

The editorial box provides a container for commentary, notes, or editorial content, with a subtle gray background and automatic "Commentary" label.

This is an editorial box for commentary or notes. It automatically adds a "Commentary" label at the top and uses a subtle gray background to distinguish it from the main content while maintaining readability.

This paragraph immediately follows the blockquote, so according to our CSS rules, it has NO indent. This prevents the visual awkwardness of an indent right after the blockquote's left border.

Now we're back to normal indented paragraphs. The rule is simple: first paragraph after a heading or blockquote gets no indent; everything else does.

Emphasis Styles

Traditional typography offers several methods of emphasis, each with its own character and purpose. We avoid bold text because it creates dark spots on the page, disrupting the even typographic color.

Instead, we use italic text for gentle emphasis. Italics suggest a shift in tone or stress without visual violence. They're perfect for foreign words, titles of works, or subtle emphasis within a sentence.

For stronger emphasis, we use small capitals. Notice how "small capitals" in the previous sentence provides emphasis without the heavy weight of bold. Small caps maintain the line's color while still drawing attention. They're traditionally used for acronyms, names, and important terms.

Compare: This is small caps emphasis versus bold text (which we've disabled). The small caps version integrates smoothly into the paragraph's texture, while bold would create a dark patch.

Working with Lists

Lists provide structure for enumerated or bulleted information. They break the flow of prose paragraphs intentionally:

  • Unordered lists use simple bullets
  • Each item has proper spacing
  • They integrate well with the overall design
  • You can emphasize within list items using italics or small caps

Ordered lists work similarly:

  1. First item in the sequence
  2. Second item, properly numbered
  3. Third item demonstrating consistent spacing
  4. Fourth item with a longer text to show how line wrapping works within list items and maintains proper alignment

After the list, paragraphs resume their normal indented flow.

Code and Technical Content

For technical writing, we need to handle code gracefully. Inline code like const example = true; is set in a monospace font with subtle background highlighting.

Larger code blocks get their own space: 

function demonstrateCode() {
  const value = "properly formatted";
  return {
    font: "monospace",
    background: "subtle gray",
    readable: true
  };
}

The code blocks maintain readability while clearly distinguishing code from prose. They use a darker background consistent with our grayscale palette.

Links and References

Links are styled subtly with an underline that appears as a border-bottom. This is more controllable than text-decoration and creates a cleaner appearance. For example, this is a link that demonstrates the styling. On hover, the underline intensifies slightly.

Multiple links like this one and that one integrate naturally into the text without overwhelming the reader with color or excessive decoration.

Tables

When data requires tabular presentation, our styling maintains the grayscale aesthetic:

Element Treatment Purpose
Paragraphs 1.5em indent Classical book style
Emphasis Italics & small caps Even typographic color
Headings No following indent Clear hierarchy

Table headers use small caps for a refined appearance, and borders are subtle gray lines that don't compete with the content.

Conclusion

This demonstration shows all the major typographic elements working together. The system is internally consistent, creating a reading experience that focuses attention on the content rather than the design.

The grayscale palette eliminates color distractions. The Georgia font provides excellent readability. The indentation rules create rhythm and hierarchy. The emphasis styles maintain even typographic color. Together, these elements produce a page that respects both the writer's craft and the reader's intelligence.

Such attention to detail may seem excessive in an age of quick consumption, but for long-form writing, it makes the difference between a document that invites sustained reading and one that encourages skimming or abandonment.
